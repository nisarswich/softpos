SoftPOS Cashier App — Mockups

High-fidelity mobile screens per the UI/UX spec, in Swich's brand theme. Each screen is a standalone, clickable HTML file — open one directly, or click through the flow. Every card below lists the internal states you can switch between using the control strip beneath the phone.

Design requirements honored: Idle (4) has no mode toggle — one action, both entry paths always live. ECR interrupt (5) is a full-bleed, flashing, action-first screen. Terminal pairing (3) never shows an IP/MAC address. QR waiting (9) shows the amount above the code with an explicit "safe to cancel" note. Network failure (10) is amber + cloud-icon, never confused with a red card decline.

4 · Dashboard & Reporting (addendum)
Adds role-based dashboard, transaction history, and reporting on top of the same login/nav shell — not a separate app. A bottom nav (Process / Dashboard / History) now lives on screen 4, 15, and 16.
Role model: Owner (all branches, branch selector shown), Branch Manager (one branch, auto-scoped, no selector), Solo/small-business owner (Dashboard + Process in the same login, no separate account type), Cashier (Process only — Dashboard/History tabs and screens 15/16/18 show an access-restricted state). Every screen in this tier has a role switcher in its mockup controls so you can see the gating react live.

5 · Settings & Appearance
App-wide light/dark toggle, plus the terminal-pairing and shift entry points consolidated in one place.
Theming: dark is the default everywhere. Toggling to light only inverts backgrounds, surfaces, and text/icon contrast — the brand gradient, logo mark, and every status color (success/decline/connectivity-failure) keep the same hue identity, re-balanced for contrast on white. Every screen's mockup controls include a quick Dark/Light switch so you can review any screen in either theme without visiting Settings first.
